# 코틀린은 자바와 무엇이 같고 무엇이 다른가?

<br />

1. 공통점
```
1) 코틀린과 자바 모두 JVM 상에서 실행된다. 이는 둘 다 자바 플랫폼에 의존하며, JVM이 제공하는 이점을 공유한다.

2) 코틀린과 자바 코드는 서로 상호 운용이 가능하다.
이는 자바 라이브러리를 코틀린에서 사용하거나, 코틀린 라이브러리를 자바에서 사용할 수 있다는 것을 의미한다.

3) 두 언어 모두 객체 지향과 함수형 프로그래밍을 지원한다.
이는 둘 다 객체 지향 프로그래밍의 특징을 가지고 있으면서도 함수형 프로그래밍의 개념을 도입할 수 있다는 것을 의미한다.
```

<br />

2. 코틀린의 장점
```
1) 자바보다 훨씬 간결하고 표현력이 뛰어난 언어이다.
코틀린은 널 안전성, 확장 함수, 데이터 클래스 등과 같은 현대적인 언어 기능을 제공하여 더 적은 코드로 더 많은 것을 표현할 수 있다.

2) 확장 함수라는 기능을 제공한다.
이를 사용하면 기존 클래스의 메서드를 수정하지 않고도 새로운 함수를 추가할 수 있다.
이는 자바에서는 인터페이스를 사용하여 비슷한 패턴을 구현할 수 있지만, 코틀린에서는 더 간편하고 직관적으로 활용할 수 있다.

3) 클래스의 기본 상속을 금지하고,
대신 인터페이스를 사용하여 코드를 구성하는 것을 권장한다.
이는 코드의 복잡성을 줄이고 유지보수를 더 쉽게 만든다.

4) 타입 추론이 강력하여 자동으로 형변환을 수행할 수 있다.
이는 코드를 더 간결하게 만들고 오류 가능성을 줄여준다.

5) null 안전 기능은 자바 버그의 일반적인 원인인 null 포인터 예외를 방지하는 데 도움이 된다.
```

<br />

3. 자바와 차이점
```
1) 자바에 비해 구문이 더 간결하여 코드를 더 쉽게 작성하고 상용구 코드의 양을 줄일 수 있다.

2) 비동기 코드를 작성하는 데 사용할 수 있는 경량 스레드인 코루틴을 기본적으로 지원하지만 자바는 지원하지 않는다.

3) 기존 클래스를 상속하지 않고도 새 함수로 확장할 수 있으므로 코드 재사용 및 기능 추가가 더 쉬워진다.
```
